# 性能計測記録

計測環境: Apple Silicon (arm64) / macOS 26 / release ビルド / decodebench

## Phase 2: DecodeKit（2026-07-03）

要件: キャッシュミス時 M262 300ms以内 / Q3 100ms以内（Q3はJPEG抽出パスが主経路のためraw経路は参考値）

### 結果（read + LJ92 + ハーフサイズの合計）

| 機種 | read (pread) | LJ92デコード | ハーフサイズ(並列) | 合計 | 判定 |
|---|---:|---:|---:|---:|---|
| M262 (24MP) | 4〜8ms | 230〜250ms | 9〜19ms | **246〜276ms** | ✔ 300ms以内 |
| Q3 18MP (参考) | 3〜7ms | 160〜168ms | 7〜12ms | **176〜194ms** | （主経路はJPEG抽出） |

- 全21サンプルでデコードエラーなし。LJ92出力は libraw と全画素一致（テストで固定）
- LJ92 は約24Mサンプル/230ms ≒ 104Mサンプル/秒（単スレッド。リスタートマーカーが無いため並列化不可）

### 最適化の履歴（計測ベース）

| 変更 | M262 LJ92 | 備考 |
|---|---:|---|
| 初版（クロージャ内ネスト関数） | 298ms | |
| ensure32一括化+modulo除去（キャプチャあり） | 525ms | **悪化**。mutable varのクロージャキャプチャでボックス化 |
| キャプチャ排除（inout引数の静的関数へ） | 270ms | |
| Huffman LUT共有（256KB→128KB）+予測器1特化 | 230ms | 両機種とも全成分同一テーブル・予測器1 |

I/O:
- mmapスライス経由のページフォールト読みは実測 ~400MB/s で26MBに50〜87ms
- pread一括読み（rawDataContiguous）で 4〜8ms に短縮 → デコードと直列でも要件内
